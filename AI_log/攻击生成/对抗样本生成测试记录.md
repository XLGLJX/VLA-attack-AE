### 1. 测试Target Manipulation Attack (TMA)

#### 运行命令
```bash
# 使用第二块GPU（GPU 1）运行测试
CUDA_VISIBLE_DEVICES=1 python VLAAttacker/TMA_wrapper.py --maskidx 0 --lr 2e-3 --server $(pwd) --device 0 --iter 10 --accumulate 1 --bs 2 --warmup 2 --tags "test" --filterGripTrainTo1 false --geometry true --patch_size "3,50,50" --wandb_project "false" --wandb_entity "test" --innerLoop 5 --dataset "libero_spatial" --targetAction 0
```

#### 预期结果
- 生成针对目标动作的对抗样本
- 输出对抗样本的相关信息

#### 实际结果
- 运行失败，遇到多个依赖缺失和代码错误

#### 遇到的问题及解决方案

**问题1：缺少dlimp模块**
- **错误信息**：`ModuleNotFoundError: No module named 'dlimp'`
- **解决方案**：安装dlimp库
  ```bash
  pip install git+https://github.com/kvablack/dlimp.git
  ```
- **说明**：dlimp安装时会自动安装TensorFlow 2.15.0和tensorflow_datasets等依赖（约475.2 MB）

**问题2：缺少tensorflow_graphics模块**
- **错误信息**：`ModuleNotFoundError: No module named 'tensorflow_graphics'`
- **解决方案**：安装tensorflow-graphics库
  ```bash
  pip install tensorflow-graphics
  ```
- **说明**：会同时安装tensorflow-addons、OpenEXR、trimesh等依赖

**问题3：缺少seaborn模块**
- **错误信息**：`ModuleNotFoundError: No module named 'seaborn'`
- **解决方案**：安装seaborn库
  ```bash
  pip install seaborn
  ```
- **说明**：会同时安装pandas等依赖（约12.8 MB）

**问题4：代码缩进错误**
- **错误信息**：`IndentationError: unindent does not match any outer indentation level` 在文件 `VLAAttacker/white_patch/appply_random_transform.py` 第43行
- **原因**：`simulation_random_patch` 方法定义使用了3个空格缩进而不是4个
- **解决方案**：修正缩进为4个空格
  ```python
  # 修改前（第43行有3个空格）
     def simulation_random_patch(self, image, patch, ...):

  # 修改后（使用4个空格）
      def simulation_random_patch(self, image, patch, ...):
  ```

**问题5：无法连接HuggingFace下载模型**
- **错误信息**：`OSError: We couldn't connect to 'https://huggingface.co' to load this file`
- **详细错误**：`Connection to huggingface.co timed out. (connect timeout=10)`
- **原因**：脚本尝试从HuggingFace下载模型 `openvla/openvla-7b-finetuned-libero-spatial`，但服务器无法访问huggingface.co
- **状态**：未解决，需要网络访问或预先下载模型到本地
- **建议解决方案**：
  1. 配置网络代理访问HuggingFace
  2. 或者预先下载模型到本地，修改代码使用本地路径
  3. 或者使用镜像站点下载模型
