# TMA 攻击运行问题总结

**日期**: 2026-03-11
**问题**: 本地模型加载失败

---

## 问题描述

使用本地模型运行 TMA 攻击时，transformers 库仍尝试联网下载配置文件。

## 已完成的修复

1. ✅ 下载模型到 `models/openvla-7b-finetuned-libero-spatial/`
2. ✅ 复制 prismatic 文件到模型目录
3. ✅ 修改 `config.json` 中的 `auto_map` 路径
4. ✅ 修改 `preprocessor_config.json` 中的 `auto_map` 路径
5. ✅ 在 `TMA_wrapper.py` 中添加 `local_files_only=True`
6. ✅ 在 `openvla_dataloader.py` 中添加 `local_files_only=True`

## 当前问题

`AutoProcessor.from_pretrained()` 内部调用 `AutoConfig.from_pretrained()` 时没有传递 `local_files_only` 参数，导致仍然尝试联网。

## 解决方案

### 方案一：设置环境变量（推荐）
```bash
export HF_HUB_OFFLINE=1
export TRANSFORMERS_OFFLINE=1
bash scripts/run_TMA.sh
```

### 方案二：修改代码直接加载
不使用 `AutoProcessor.from_pretrained()`，而是直接实例化：
```python
from prismatic.extern.hf.processing_prismatic import PrismaticProcessor
from prismatic.extern.hf.configuration_prismatic import OpenVLAConfig

config = OpenVLAConfig.from_pretrained(vla_path, local_files_only=True)
processor = PrismaticProcessor.from_pretrained(vla_path, local_files_only=True)
```

### 方案三：使用代理下载（已测试可用）
```bash
export HTTP_PROXY="http://127.0.0.1:7890"
export HTTPS_PROXY="http://127.0.0.1:7890"
bash scripts/run_TMA.sh
```

---

## 建议

使用**方案一**（设置离线环境变量）最简单可靠。
